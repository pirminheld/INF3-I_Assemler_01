Assembler ist nicht schwer, nur sehr nah an der Maschine

In Java schreibt ihr eher Absichten. In Assembler gebt ihr der CPU sehr kleine Einzelschritte.

Die wichtigste Idee

Eine CPU kann nicht direkt verstehen, was ein Array, eine Klasse oder eine Schleife in Java bedeutet. Sie kennt kleine Speicherplaetze, Zahlen und Befehle wie: kopiere, addiere, springe, vergleiche.

Register Kleine Arbeitsfaecher in der CPU: A, R0, R1, R2 ...
Rechenwerk Fuehrt Befehle aus: INC, ORL, RL, RR, CJNE ...
Ausgaenge Ports wie P2 steuern LEDs, Anzeige oder andere Bauteile.

Java-Vergleich

int a = 1;
a = a + 1;

Java versteckt viele Details: Wo liegt `a`? Welche CPU-Befehle entstehen daraus? Das uebernimmt der Compiler.

Assembler-Vergleich

MOV A, #1
INC A

Hier seht ihr die Maschine direkt: Lade den Wert 1 in das Register A. Erhoehe A um 1.

Merksatz

Assembler ist eine sehr direkte Sprache fuer Prozessorbefehle. Jeder Befehl veraendert meistens genau ein kleines Stueck Zustand: ein Register, ein Bit, einen Speicherplatz oder den Programmzaehler.

Bits, Hex und Ports

Der Simulator arbeitet oft mit 8 Bit. Ein Byte kann Werte von 0 bis 255 speichern.

Interaktiver Umrechner

Gib eine Zahl ein und beobachte, welche LEDs an Port P2 leuchten wuerden.

Binaer 00101010b Hex 2Ah
P2.7P2.6P2.5P2.4 P2.3P2.2P2.1P2.0

Schreibweisen

Im Beispielcode findet ihr drei wichtige Schreibweisen:

MOV P2, #255      ; dezimal
MOV P2, #11111111b ; binaer
DB 7Eh            ; hexadezimal

Das `#` bedeutet: Es kommt ein direkter Wert. Ohne `#` wird oft der Inhalt eines Registers oder Ports gemeint.

Mini-Aufgabe

Welche Zahl schaltet nur P2.0 und P2.3 ein?

Tipp: P2.0 ist 1, P2.3 ist 8.

Erste Befehle lesen

Ihr muesst am Anfang nicht alle Befehle kennen. Ein kleiner Wortschatz reicht fuer die ersten Programme.

MOV

MOV A, #5
MOV P2, A

Kopiert Werte. Erst kommt das Ziel, dann die Quelle: `MOV Ziel, Quelle`.

INC und DJNZ

INC R4
DJNZ R0, warte1

`INC` erhoeht um 1. `DJNZ` verringert um 1 und springt, solange das Ergebnis nicht 0 ist.

Spruenge

main:
SJMP main

Labels sind Sprungmarken. `SJMP main` springt zur Marke `main` zurueck.

Bitbefehle

JB P2.7, rotateright
JNB P3.2, ende

`JB` springt, wenn ein Bit 1 ist. `JNB` springt, wenn ein Bit 0 ist. Damit werden Schalter und Taster abgefragt.

Unterprogramme

LCALL wait
RET

`LCALL` ruft ein Unterprogramm auf. `RET` kehrt wieder zurueck. Das ist entfernt vergleichbar mit einem Methodenaufruf in Java.

Kommentare schreiben

MOV A, #5       ; A bekommt den Wert 5
INC A           ; A wird um 1 erhoeht

Alles rechts vom Semikolon `;` ist Kommentar. Der Simulator fuehrt Kommentare nicht aus. Gute Kommentare erklaeren den Zweck einer Zeile, nicht nur den Befehl noch einmal.

Befehlsuebersicht dieser Einfuehrung

MOV Ziel, QuelleKopiert einen Wert, zum Beispiel `MOV A, #5` oder `MOV P2, A`.
INC RegisterErhoeht einen Wert um 1, zum Beispiel `INC R4`.
DJNZ Register, labelVerringert um 1 und springt, solange das Ergebnis nicht 0 ist.
SJMP labelSpringt immer zu einer Marke. `SJMP main` erzeugt oft eine Dauerschleife.
JB bit, labelSpringt, wenn das Bit 1 ist. Beispiel: `JB P2.7, rechts`.
JNB bit, labelSpringt, wenn das Bit 0 ist. Beispiel: `JNB P3.2, warten`.
CJNE Wert, Vergleich, labelVergleicht zwei Werte und springt, wenn sie ungleich sind.
LCALL labelRuft ein Unterprogramm auf.
RETBeendet ein Unterprogramm und kehrt zurueck.
RL A / RR ARotiert die Bits in A nach links oder rechts.
ORL Ziel, QuelleVerknuepft Bits mit ODER. Ein Bit wird 1, wenn eines der beiden Bits 1 ist.
DB WerteLegt feste Tabellenwerte im Programmspeicher ab.
MOVC A, @A+DPTRLiest einen Wert aus einer Tabelle. A ist der Index, DPTR zeigt auf die Tabelle.
ADD A, QuelleAddiert einen Wert zum Akkumulator A.
DEC RegisterVerringert einen Wert um 1.
DIV ABTeilt A durch B. Das Ergebnis landet in A, der Rest in B.
SETB bitSetzt ein einzelnes Bit auf 1.
CLR bitSetzt ein einzelnes Bit auf 0.
CPL bitKehrt ein Bit um: aus 1 wird 0, aus 0 wird 1.
ORG AdresseLegt fest, an welcher Programmadresse die folgenden Befehle stehen. Wichtig bei Interrupts.
RETIBeendet ein Interrupt-Unterprogramm.
label:Markiert eine Stelle im Programm, zu der gesprungen werden kann.
#WertDirekter Wert. `#5` bedeutet die Zahl 5, nicht den Inhalt von Adresse 5.
@R0Indirekter Zugriff: Verwende die RAM-Adresse, die in R0 steht.
;Startet einen Kommentar bis zum Ende der Zeile.

Lesestrategie fuer Anfaenger

            
1. Labels suchen 2. MOV-Befehle markieren 3. Spruenge verfolgen 4. Ports erkennen

            
Wenn ein Programm lang wirkt: Nicht von oben bis unten "auswendig verstehen". Sucht zuerst die Hauptschleife, dann Unterprogramme wie `warte`, dann Ein- und Ausgaenge wie `P1`, `P2`, `P3.2`.

Naechste sinnvolle Schritte

            
Nach dieser Einfuehrung ist der beste Einstieg ein sehr kleines Programm, das sofort sichtbare Wirkung hat.

            
MOV P2, #00000001b
ende:
SJMP ende

            

              
1. LED gezielt einschaltenAendere den Wert hinter `#` und beobachte P2. Teste `#1`, `#2`, `#4`, `#8` und `#255`.

              
2. Wert in A vorbereitenSchreibe erst `MOV A, #1` und danach `MOV P2, A`. So wird sichtbar, wozu Register dienen.

              
3. Bits verschiebenFuege `RL A` oder `RR A` ein und kopiere danach wieder nach P2.

              
4. Schleife bauenNutze ein Label und `SJMP`, damit ein Ablauf wiederholt wird.

              
5. Warten verstehenNutze `DJNZ`, um eine kuenstliche Verzoegerung zu bauen.

              
6. Taster abfragenNutze `JB` oder `JNB`, um auf P3.2 oder P3.3 zu reagieren.

              
7. Tabelle lesenNutze `DB`, `DPTR` und `MOVC`, wenn mehrere feste Ausgabemuster gebraucht werden.

              
8. Kommentare ergaenzenSchreibe hinter wichtige Zeilen mit `;`, was der Schritt fuer das Programm bedeutet.
